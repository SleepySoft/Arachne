# ============================================================
# RESOURCE
# 表示能够参与产业活动，或由产业活动产生的有形、无形资源。
# ============================================================

RESOURCE:

- material
  # 物料。
  # 包括原料、零部件、中间品、成品、辅料、能源载体等有形对象。

- service
  # 服务。
  # 表示可被提供、使用或组合的服务能力，如物流、检测、维修、代工。

- right
  # 权利或授权。
  # 表示使用、实施、访问或经营某对象的权利，如软件许可、专利实施权。

- information
  # 信息资源。
  # 包括数据、软件、设计图纸、技术资料、工艺配方、分析结果等。

- qualification
  # 认证、许可或资质。
  # 表示开展某类活动或证明符合某项要求的资格性资源。

- other
  # 暂时无法归入以上类别的其他产业资源。


# ============================================================
# INPUT ROLE
# 表示 RESOURCE 在当前 Action 中扮演的输入角色。
# Role 是局部语义，同一 RESOURCE 在不同 Action 中可以具有不同角色。
# ============================================================

input_role:

# ------------------------------------------------------------
# 主体输入
# ------------------------------------------------------------

- feedstock
  # 主体原料或主要加工对象。
  # 通常经过转化、加工、分离或混合成为主要输出。

- component
  # 组成部件。
  # 经组合后成为输出资源的一部分，通常保留可识别的结构或功能身份。

- additive
  # 添加物。
  # 加入主体资源并通常进入输出，但不是主要加工对象。


# ------------------------------------------------------------
# 工艺参与
# ------------------------------------------------------------

- process_material
  # 工艺材料。
  # 参与活动并可能被消耗，但通常不是主要输出的主体来源。

- catalyst
  # 催化或促进资源。
  # 促进活动发生，但通常不是主要输出的物质或内容来源。

- energy
  # 能源。
  # 为活动提供电、热、光、燃料或其他形式的能量。

- carrier
  # 载体。
  # 用于承载、支撑、输送或提供介质，本身通常不是主要输出。

- tool
  # 工具或工装。
  # 活动依赖的刀具、模具、设备、平台、软件工具等。

- packaging
  # 包装资源。
  # 用于封装、保护、运输或形成交付形态的资源。


# ------------------------------------------------------------
# 通用非物质参与
# ------------------------------------------------------------

- subject
  # 活动作用的对象。
  # 适用于维修、检测、认证、运输、软件定制等不宜称为原料的输入。

- basis
  # 活动依据。
  # 为活动提供知识、规则、技术或信息依据，如标准、图纸、数据、专利。

- requirement
  # 资格或权利要求。
  # 表示开展该活动通常依赖的许可、资质、授权或其他前提性资源。
  # 当前仅表示定性依赖，不表示严格且不可替代的必要条件。

- other
  # 确定参与 Action，但暂时无法归入以上角色的输入资源。


# ============================================================
# ACTION
# 表示资源之间的一次类型级活动或转换节点。
# Action 是流程中的局部节点，可以引用可复用的工艺或工艺组。
#
# Action 不是批次执行实例。
# 同一工艺出现在不同流程中时，应建立不同 Action，并引用同一工艺。
# ============================================================

Action:

- transform
  # 转化。
  # 输入被转化为语义上不同的资源，原主体身份通常不再保持。
  # 例如：矿石变为金属、原始数据变为分析结果。

- combine
  # 组合。
  # 多个资源被组合为一个更高层资源。
  # 例如：部件组装为电脑、多项服务组合为解决方案。

- separate
  # 分离。
  # 一个资源或资源集合被拆分、分离、提取或解构为多个输出。
  # 例如：原油分馏、废旧设备拆解、数据集拆分。

- modify
  # 修改或加工。
  # 主体连续性大体保持，但其形态、性质、配置或内容发生改变。
  # 例如：机械加工、软件定制、设备维修。

- deliver
  # 提供、开放或授予。
  # 形成可被使用的服务、权利或信息资源。
  # 例如：提供检测服务、开放数据访问、授予软件许可。
  #
  # 不建议用于普通生产流程中的中间物料流转；
  # 常规物料加工和生产应优先使用 transform、combine 或 modify。

- assess
  # 检测、判断或评价。
  # 对资源进行测试、审核、评估或认证，并产生分类、信息或资质结果。
  # 例如：芯片测试、产品认证、技术审核、风险评估。

- other
  # 暂时无法归入以上类别的活动。


# ============================================================
# OUTPUT ROLE
# 表示 RESOURCE 在当前 Action 输出中的结果性质。
# Role 是相对于当前 Action 或 Flow 的局部语义，
# 不是 RESOURCE 自身永久不变的分类。
# ============================================================

output_role:

- primary_result
  # 主要结果。
  # Action 的主要目标输出，可以是物料、服务、权利、信息或资质。

- co_result
  # 联合结果。
  # 与其他主要结果共同产生，并具有相对独立用途或价值的输出。

- intermediate
  # 中间结果。
  # 在当前流程中继续作为后续 Action 输入的资源。
  # 同一资源在其他流程边界下可能是 primary_result。

- byproduct
  # 副产结果。
  # 不是 Action 的主要目标，但具有潜在用途或价值的伴生产出。

- scrap
  # 可回收余料或残次资源。
  # 通常仍具有回收、再利用或进入其他流程的潜在价值。

- waste
  # 废弃物。
  # 在当前流程语境中无计划用途，通常需要处理或处置的输出。

- emission
  # 排放物。
  # 活动向环境或处理系统释放的气体、液体、颗粒、热等资源。

- recovered_resource
  # 回收资源。
  # 经拆解、分离、再生或回收活动重新获得的可利用资源。

- other
  # 确定为 Action 输出，但暂时无法归入以上角色的结果。

# ============================================================
# 基本建模原则
# ============================================================

1. RESOURCE.type 表示“资源是什么”。

2. input_role 表示“资源如何参与当前 Action”。

3. Action.type 表示“当前活动主要对资源做了什么”。

4. output_role 表示“资源在当前输出中处于什么位置”。

5. Role 是 Action 局部语义，不是 RESOURCE 的永久属性。

6. 工艺描述通用、稳定的资源参与关系。

7. Flow 中的 Action 描述具体路线的主要资源输入与输出。

8. 多个流程可以复用同一工艺，但不应直接复用同一个 Action。

9. 工艺关系可以由 Action 引用继承，避免复制到每个 Action。

10. 当前模型只表达定性关系：
    不表达数量、比例、可选性、物料状态、参数匹配和严格前置条件。

11. requirement 只表示关联的资格或权利要求：
    在没有更强证据时，不应推导为绝对必要条件。

12. 基于路径得到的结果只表示结构可达或潜在关联：
    不直接表示技术可行、精确物质归属或必然依赖。
